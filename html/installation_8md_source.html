1 # Installation {#installation}
2 
3 After having installed all necessary [dependencies](@ref dependencies), the library can be compiled out of source and installed using _cmake_. In the following we assume a build folder named 'build' will be used. The following terminal commands will allow for a build with the default settings enabled (for more options see [options](@ref options)):
4 
5 ~~~.bash
6 cmake ..
7 make resources
8 make
9 ~~~
10 
11 To install the library to your system, use
12 
13 ~~~.bash
14 make install
15 ~~~
16 
17 Options {#options}
18 ------------------
19 In its current state, the main features of HyPro can be configured via _cmake_ or its gui _ccmake_, which can be invoked by (from the build folder):
20 
21 ~~~.bash
22 ccmake ..
23 ~~~
24 
25 The currently available options are:
26 * HYPRO_LOGGING - Allows to completely enable/disable all logging macros (default: ON).
27 * HYPRO_USE_COTIRE - Enables/disables the usage of [cotire](https://github.com/sakra/cotire), which allows for faster builds (default: OFF).
28 * HYPRO_USE_OPENMP - Enables/disables the usage of OpenMP in Eigen3 (default: OFF).
29 * HYPRO_USE_PPL - Enables/disables the PPL-wrapper class. Note that PPL has to be installed to use this feature (default: OFF).
30 * HYPRO_USE_SMTRAT - Enables/disables the usage of SMT-RAT as a secondary linear optimization backend. Note that SMT-RAT has to be installed to use this feature (default: OFF).
31 * HYPRO_USE_SOPLEX - Enables/disables the usage of SoPlex as a secondary linear optimization backend. Note that SoPlex has to be installed to use this feature (default: OFF).
32 * HYPRO_USE_Z3 - Enables/disables the usage of Z3 as a secondary linear optimization backend. Note that Z3 has to be installed to use this feature (default: OFF).
33 * STATICLIB_SWITCH - Allows to compile a static-linked version of HyPro (default: OFF).
34 * CREATE_DOC - Allows to locally build this documentation (default: OFF).

38 Examples
39 --------

41 Each provided example file comes with its own target, which can be build by using the examples name, i.e. use
42

44 make example_zonotope
45 ~~~
