266  /**
267  * extrapolates the current DBM by using the ExtraLU Method. A description
268  * can be found in: Behrman, Bouyer "Lower and Upper Bounds in Zone Based Abstractions of Timed Automata"
269  *
270  * Lower and upper bounds are defined as follows:
271  * The maximal lower bound denoted L(x) (resp. maximal upper bound U(x)) of clock x is the maximal
272  * constant k such that there exists a constraint x>k or x>=k (resp x<k or x<=k) in a guard
273  * of some transition or in an invariant of some location of the automaton. If such a constant
274  * does not exist we set L(x) (resp. U(x)) to -INFTY. (Here: DBMEntry with BOUNDTYPE::INFTY)
275  *
276  * @param LBounds List of DBMEntries describing the maximal lower Bound for each variable i.e. the largest constant the variable is compared against. Note that the BOUND_TYPE has no effect unless it is infinity which denotes that the variable is not compared against any constant (i.e. is interpreted as -INFTY) The Bound for the 0 clock is 0.
277  * @param UBounds List of DBMEntries describing the maximal upper Bound for each variable i.e. the largest constant the variable is compared against. Note that the BOUND_TYPE has no effect unless it is infinity which denotes that the variable is not compared against any constant (i.e. is interpreted as -INFTY) The Bound for the 0 clock is 0.
278  * @return The extrapolated DBM
279  */
