1 /*
2  * SupportFunctionNew.h
3  *
4  * Class representing a SupportFunction.
5  * Here, a SupportFunction is a tree where a new node is set as the parent of the current root, thus growing at the root.
6  * The SupportFunction itself knows and owns the root node, which in turn knows its children and so on.
7  * Note that the pointer to the root node will not be shifted when a new node is added.
8  * Through this, it is possible to query an evaluation of the operations from each node in the tree.
9  * Each leaf node in this tree holds a representation of a state, i.e. a box, a V-polytope, etc.
10  * Each node that is not a leaf represents an operation that is performed on its children.
11  *
12  * @author Stefan Schupp
13  * @author Phillip Tse
14  */

217  /*
218  * traverse() explores all nodes beginning from the root and performs the given functions while going down/up.
219  * Three functions are needed: transform, compute and aggregate.
220  * - transform: will be called by every node and mainly transforms additional parameters
221  * - compute: will only be called once by leaf nodes and computes a result for unmodified representations such as box, V-polytope, etc.
222  * - aggregate: will only be called once by all non leaf nodes and accumulates the results of its children
223  *
224  * Order of calling:
225  * 1) From top to bottom, all nodes call transform
226  * 2) When arriving at the lowest level, leaf nodes first call transform, then call compute
227  * 3) From bottom to top, each node that is not a leaf node calls aggregate
228  *
229  * There are only four possibilities of how void functions and / or functions without parameters can be combined, such that the stack operations
230  * made during the actual traversal still remain valid:
231  * - Result = void, Params = void: "void transform()" && "void compute()" && "void aggregate()" && no initParams
232  * - Result = int, Params = void: "void transform()" && "int compute()" && "int aggregate(vector<int>)" && no initParams
233  * - Result = void, Params = int: "int transform(int)" && "void compute(int)" && "void aggregate(int)" && initParams
234  * - Result = int, Params = int: "int transform(int)" && "int compute(int)" && "int aggregate(vector<int>, int)" && initParams
235  * If the parameter type does not exist (so, Rargs... = empty set), then additional parameters can be left out in all three functions,
236  * and if parameters would be returned, void is returned instead (i.e. transform).
237  * If the result type is void, then the vector to be aggregated for the aggregate function can be left out.
238  * Below, the four possibilites are implemented, functions returning void or having no additional parameters are wrapped into functions that return
239  * empty parameters & receive one additional parameter.
240  */
241
